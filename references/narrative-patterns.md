# Narrative Patterns

Select a pattern based on the decision, not the topic.

## Decision / purchase

`why change -> why now -> expected value -> controlled risks -> why us -> requested decision`

## Company capability

`customer context -> capability thesis -> integrated capability system -> differentiated proof -> representative outcomes -> cooperation entry`

Avoid beginning with company history unless credibility itself is the decision barrier.

## Project solution

`customer situation -> critical problems and causes -> design principles -> solution blueprint -> scenarios and workflow -> implementation and governance -> value and proof -> next step`

## Product solution

`market or business shift -> user problem -> product judgment -> capability architecture -> representative workflows -> deployment and operations -> proof -> adoption path`

## Technical qualification

`evaluation standards -> current gap -> architecture -> integration -> security and governance -> reliability and operations -> validation -> boundaries and risks`

## Business co-creation

`current job/workflow -> pain and impact -> future job/workflow -> scenarios -> human-system collaboration -> measures -> pilot design`

## Progress review

`original objective -> current progress -> demonstrated outcomes -> issues and causes -> adjustment -> next phase -> decisions/support needed`

## Existing-deck optimization

First classify the need as local refinement, chapter reorder, narrative rebuild, or audience-version split. Preserve useful evidence and assets; do not retain pages merely because they already exist.

## Message map

Use one core claim, no more than three supporting reasons, evidence for each reason, and one requested action. Every chapter must perform a distinct persuasion task.
